# 🤖 Thai Trade Bot — Backtest Starter

ลองทำเล่นๆ · เงินน้อยๆ · **พิสูจน์ด้วย backtest ก่อน ค่อยเงินจริง**
(ของ [[Nattapon_Hongkan]] — เล่นในกรอบ Sandbox ≤10% เท่านั้น)

> ⚠️ บอต ≠ เครื่องพิมพ์เงิน · Retail algo ส่วนใหญ่แพ้ buy-and-hold · ทำเป็น "โปรเจคเรียนรู้ + ตัดอารมณ์" ไม่ใช่ทางรวย

---

## 🪜 แผน 3 ขั้น (ห้ามข้าม)
```
1. BACKTEST (ตอนนี้)  → รันบนข้อมูลอดีต ดูว่ากลยุทธ์เวิร์กไหม  ← เงิน 0 บาท
2. PAPER TRADE        → รันสด เงินปลอม 1-3 เดือน
3. LIVE จิ๋ว          → เงินจริงก้อนเล็ก + kill switch ผ่าน Settrade Open API
```
**เกณฑ์ผ่านไปขั้นถัดไป:** กำไรสม่ำเสมอ + Max Drawdown รับได้ + win rate/RR สมเหตุผล
(ถ้า backtest ยังไม่ผ่าน = อย่าเอาเงินจริงเข้า)

---

## 🚀 วิธีรัน
```bash
pip install -r requirements.txt

# ทดสอบกลยุทธ์ MA cross กับหุ้นไทย
python backtest.py --symbol PIMO.BK --years 5 --fast 20 --slow 50

# ลองหลายตัว / ปรับพารามิเตอร์
python backtest.py --symbol SELIC.BK --fast 10 --slow 30
```

อ่านผล:
- **เทียบ Buy & Hold เสมอ** — ถ้าบอตแพ้การถือเฉยๆ = กลยุทธ์ยังไม่คุ้ม
- ดู **Max Drawdown** (เจ็บสุด) + **จำนวนไม้** (ยิ่งเยอะยิ่งโดนค่าธรรมเนียม)

---

## 📊 กลยุทธ์เริ่มต้น (backtest.py)
**SMA Crossover** (long-only — หุ้นไทย short ยาก):
- เส้นเร็ว (fast) ตัดขึ้นเหนือเส้นช้า (slow) → ซื้อ
- ตัดลง → ขาย
- รวม **ค่าธรรมเนียม ~0.2%/ข้าง** ในการคำนวณ (ไม่งั้นหลอกตัวเอง)

> เริ่มง่ายๆ ก่อน · อย่าเพิ่งใส่ indicator 10 ตัว (overfit)

---

## 🔌 ขั้น LIVE (อนาคต — ยังไม่ทำตอนนี้)
- **Settrade Open API** = สั่งซื้อ-ขายอัตโนมัติได้ (ต้องขอเปิดสิทธิ์กับโบรก)
- ต้องมี: **kill switch**, max loss/วัน, position sizing, logging
- ทดสอบใน paper ของ Settrade ก่อนเสมอ

---

## ⚠️ กับดักที่ต้องระวัง
- **Overfit**: ปรับพารามิเตอร์จนสวยในอดีต → ของจริงเจ๊ง (แก้: ทดสอบช่วงเวลาอื่นด้วย / out-of-sample)
- **Survivorship / ราคาปรับ**: ใช้ข้อมูล auto-adjust
- **สภาพคล่อง**: หุ้นเล็ก backtest สวยแต่ซื้อจริงไม่ได้ราคานั้น
- **yfinance หุ้นไทย** อาจดีเลย์/ข้อมูลไม่ครบ → ของจริงใช้ข้อมูล Settrade
